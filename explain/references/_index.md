## File Summaries

### process.md
The six Feynman steps in full detail. Includes the opening format, how to expose the flaw in common understanding, the analogy rules, the "curious 10-year-old" clarification, and the closing line rules. This is the primary reference during explanation.

### context-gathering.md
Three-step protocol for gathering source material before explaining. Covers codebase search, web search, model knowledge, and the exact conditions under which to stop and ask the user. Includes guidance on how to ask specifically rather than vaguely.

### adaptive-explanation.md
How to read understanding signals (strong, partial, confused, over-explained), what to do when something doesn't land (diagnose before re-explaining), how to check in without being patronising, and how to update the user's persistent preference file after each session. This is what makes the skill improve over time for this specific user.

### programming-explanation.md
How to make programming explanations more digestible: name the pattern before dissecting it, show real code before describing it, explicitly name and replace the wrong mental model, separate "what / why / when" questions, trace execution for a concrete case, layer one concept per paragraph, and close with a specific mistake the right understanding prevents.

### learning-science.md
The cognitive science that grounds the technique — active recall, elaborative explanation, metacognition, chunking, transfer learning. Covers the domains where Feynman style works best, the documented exceptions (memorisation, tacit knowledge, abstract mathematics, emotional situations), the over-simplification trap, and all research sources.
